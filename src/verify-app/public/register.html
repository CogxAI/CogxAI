Get started with CogxAI

Pick a deployment mode. You can change it anytime.

Local Zero config
Everything stays on your device. No accounts, no keys, completely offline. Ideal for testing or when privacy matters most.
npx cogxai mcp-serve --local
Self-hosted Your infra
Connect your own Supabase and Voyage AI credentials. Complete control over your data. No CogxAI account required.
npm install cogxai && npx cogxai setup
Cloud Fastest
Let CogxAI handle the infrastructure. Sign up, receive an API key, and begin storing memories immediately.
curl -X POST https://cogxai.org/api/cortex/store ...

Local mode

No credentials required. Your machine handles everything.

For Claude Desktop / Cursor (MCP)
# One command, done
npx cogxai mcp-install --local
Or add manually to Claude Desktop config
{
  "mcpServers": {
    "cogxai": {
      "command": "npx",
      "args": ["cogxai", "mcp-serve", "--local"]
    }
  }
}
For Node.js / TypeScript
# Install
npm install cogxai

# Use
import { Cortex } from 'cogxai';
const brain = new Cortex({ local: true });
await brain.init();
await brain.store({ content: 'Hello!', type: 'episodic' });
const memories = await brain.recall({ query: 'hello' });

All data lives in ~/.cogxai/memories.json. GTE-Small embeddings (~30MB) download automatically the first time you run it.

Self-hosted mode

No CogxAI account necessary. You plug into your own infrastructure directly.

What you need
Supabase — stores memories + vector search (free tier works)
Voyage AI — embedding generation (voyage-4-large)
Anthropic — powers dream cycles + reflection (optional)
Setup
# Install and run the setup wizard
npm install cogxai
npx cogxai setup
Or configure manually
import { Cortex } from 'cogxai';

const brain = new Cortex({
  supabase: {
    url: process.env.SUPABASE_URL,
    serviceKey: process.env.SUPABASE_KEY,
  },
  embedding: {
    provider: 'voyage',
    apiKey: process.env.VOYAGE_API_KEY,
    model: 'voyage-4-large',
    dimensions: 1024,
  },
  agentId: 'my-agent',
});
await brain.init();
